# Merch Bot — Водопады

Телеграм-бот для учёта продаж мерча группы.

## Структура

```
bot.py           — основной бот, все хэндлеры
database.py      — SQLite: позиции, продажи, мероприятия
channel_stats.py — обновление трёх сообщений в канале статистики
requirements.txt
Procfile         — для Railway
```

## Переменные окружения

| Переменная   | Что это                                              |
|--------------|------------------------------------------------------|
| `BOT_TOKEN`  | Токен бота от @BotFather                             |
| `CHANNEL_ID` | ID канала статистики, например `-1001234567890`      |
| `DB_PATH`    | Путь к SQLite-файлу (опционально, по умолчанию `merch.db`) |

## Деплой на Railway

1. Создай аккаунт на [railway.app](https://railway.app)
2. New Project → Deploy from GitHub repo (залей папку на GitHub)
   — или — New Project → Empty project → Add service → GitHub repo
3. В разделе Variables добавь `BOT_TOKEN` и `CHANNEL_ID`
4. Railway сам подхватит `Procfile` и запустит `python bot.py`

> На Railway бесплатный тариф даёт ~500 часов/месяц — для бота хватит.
> Для персистентного хранения БД подключи **Railway Volume** (в настройках сервиса → Volumes → Mount Path: `/data`) и поставь `DB_PATH=/data/merch.db`

## Подготовка канала

1. Создай приватный канал (или группу) для статистики
2. Добавь бота в канал как **администратора** с правом публиковать сообщения
3. Узнай ID канала: перешли любое сообщение из канала боту @userinfobot — он покажет ID
4. ID канала начинается с `-100...` — вставь его в `CHANNEL_ID` целиком со знаком минус

## Использование

`/start` или `/menu` — главное меню

**Продажа:**
→ «Продал» → выбери мероприятие → выбери позицию → подтверди

**Остатки:**
→ «Остатки» — список всех позиций с количеством
→ «Обновить остаток» — выбери позицию, введи новое число

**Добавить позицию:**
→ «Добавить позицию» → название → цвет → размер → цена → остаток

**Добавить мероприятие:**
→ «Добавить мероприятие» → введи название

**Канал статистики** обновляется автоматически после каждой продажи и изменения остатков. Три сообщения:
- Выручка (итого + по мероприятиям)
- Остатки и продажи по позициям
- Последние 10 продаж
